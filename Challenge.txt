Required Functionalities
1. Process Control API
● POST /process/start - Start a new analysis process
● POST /process/stop/{process_id} - Stop a specific process
● GET /process/status/{process_id} - Query the status of a process
● GET /process/list - List all processes and their states
● GET /process/results/{process_id} - Get analysis results

2. Processing System
● Load text files from a specific folder
● Process documents in batches (batch processing)
● Extract basic statistics: word count, lines, characters
● Identify most frequent words
● Generate a content summary

3. System States
● PENDING - Process created but not started
● RUNNING - Processing in progress
● PAUSED - Process temporarily paused
● COMPLETED - Process finished successfully
● FAILED - Process terminated with errors
● STOPPED - Process manually stopped

4. Results Storage
● Persist process states
● Store analysis results
● Maintain activity logs

5. Minimum Requirements
● Functional REST API with specified endpoints
● Real asynchronous processing (not simulated)
● Concurrent state management
● Input validation and error handling
● API documentation

6. Test Data

demonstrate system functionality.
